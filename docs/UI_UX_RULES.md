# UI UX Rules

## Layout

- Use one primary workspace per screen.
- Keep page sections full-width bands or disciplined panels, not nested floating cards.
- Prefer dense, scannable layouts for operational work.
- Preserve stable dimensions for toolbars, tiles, counters, and sidebars.

## Typography

- Use a clear hierarchy with compact headers inside tools.
- Keep labels short and action-oriented.
- Do not use oversized marketing type inside working screens.

## Spacing

- Leave more room around primary data than around decoration.
- Use consistent padding across cards, drawers, and dialogs.
- Avoid crowded tool clusters.

## Workflow

- Show the current stage, the next required action, and the output that will be produced.
- Make stage completion visible.
- Keep navigation aligned to the real production flow, not just to screen names.

## Visual Language

- Favor premium restraint over busy ornament.
- Use contrast and spacing to show importance.
- Keep render, elevation, material, and finance surfaces visually related so they feel like one product.
