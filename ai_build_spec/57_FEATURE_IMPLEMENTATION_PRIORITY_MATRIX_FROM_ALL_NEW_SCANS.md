# 57 — Feature Implementation Priority Matrix from All New Scans

## Purpose

This document prioritizes all newly observed competitor feature classes into:
- build now
- build next
- build later
- reject/defer

---

## 1. Build Now

These features directly strengthen StudioOS’s core value proposition and should be implemented first.

### Workflow surface
- Smart Project end-to-end polish
- Quick Generate concept mode
- Quick Layout create/edit mode
- Design Product workspace
- Photo Edit session foundation
- workflow-first dashboard
- tool hub / AI Studio entry system

### Design core
- overlay editing maturity
- room/wall/opening review tools
- faster scene promotion from reviewed plan
- room-specific templates and references
- category-specific preview assets

### Render/drawings/BOM continuity
- memory-guided render suggestions
- render angle workflows
- walkthrough path serialization
- stronger elevation packs
- drawing pack assembly
- BOM preview maturity

### Presentation / sales layer
- client presentation pack
- room boards and angle boards
- render shortlisting and compare flows

---

## 2. Build Next

These features are strong multipliers but depend on the core being stable.

### Catalog / brand / marketplace depth
- catalog entities
- vendor/brand entities
- finish libraries
- product/material publish flows
- quote request basis

### Agent orchestration
- plan review agent
- style/material agent
- render recommendation agent
- documentation agent
- budget optimization agent

### Product intelligence
- product family generation
- product drawing workflows
- stronger BOM and takeoff logic

### Real-estate readiness
- unit presentation packages
- finish package toggles
- buyer-ready dashboards

---

## 3. Build Later

These are valuable but should not distract from the main product maturation path.

### Manufacturing/ERP integration depth
- inventory sync
- label printing workflows
- shop-floor traceability integration
- machine bot integrations

### Smart-home integration
- automation packages
- device recommendations
- future scene presets for smart living

### Larger AEC/BIM intelligence
- deeper IFC orchestration
- building-scale clash/reports beyond interior-first scope

---

## 4. Reject or Defer Hard

Do not do these early:
- restart repository from scratch
- build separate greenfield AURA platform
- overbuild K8s/GPU infra
- overbuild enterprise multi-mode before designer mode is excellent
- copy service-studio brochure features as if they are product depth

---

## 5. Final Rule

Use this matrix whenever new ideas appear:

### Build now if
- it strengthens the geometry-first design core
- it improves plan-to-scene-to-output continuity
- it reduces designer repetition
- it improves render/drawing/BOM/approval usability

### Build later if
- it is useful but depends on broader product maturity

### Reject/defer if
- it adds complexity without improving the core operating loop
