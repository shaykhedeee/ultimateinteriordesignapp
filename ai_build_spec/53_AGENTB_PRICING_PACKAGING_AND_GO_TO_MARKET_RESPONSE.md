# 53 — Agent B Pricing, Packaging, and Go-to-Market Response

## Purpose

This document captures pricing and packaging lessons from Agent B’s public surface and translates them into StudioOS GTM guidance.

---

## 1. Pricing Lessons Observed

Public signals from Agent B suggest:
- low-friction trial
- solo designer tiers
- pro tier with more renders and video
- unlimited higher tier
- brand / enterprise pricing for multi-seat and catalog workflows
- workflow bundles instead of per-feature fragmentation

---

## 2. Packaging Recommendation for StudioOS

## 2.1 Solo / Designer tiers

### Starter
- limited monthly renders
- unlimited basic 2D layout
- plan upload + calibration + review
- limited drawing/export outputs

### Pro
- more render quota
- walkthroughs
- advanced material tools
- elevation pack generation
- BOM previews
- client presentation exports

### Studio / Unlimited
- unlimited render usage or high fair-use ceiling
- multi-project management
- multi-seat collaboration later
- advanced commercial / production workflows

---

## 2.2 Brand / Manufacturer tiers

### Brand Starter
- product library onboarding
- finish library
- product drawings and BOM
- limited seat count

### Brand Pro
- larger library
- branded catalog / workspace
- custom quote logic
- vendor workflows

### Enterprise
- tailored integrations
- account management
- white-label / custom workflows

---

## 2.3 Developer / Real Estate tiers

### Developer Basic
- unit visualization
- finish package presentation
- buyer-facing presentation mode

### Developer Enterprise
- project dashboarding
- unit portfolio management
- interactive customization
- branded sales workflows

---

## 3. StudioOS Differentiated GTM Angle

StudioOS should position itself not just as a visualization system, but as:

### “The operating system for modular interior design and production continuity.”

That means the strongest sales messages should be:
- plan to production continuity
- editable geometry truth
- faster repeated design work
- Indian modular depth
- elevations and BOM from approved scene
- reduction in repetitive designer effort

---

## 4. Feature Gating Recommendation

Do not gate only by random tool count.
Gate by workflow power.

### Suggested gating examples
- Starter: concept + layout + limited render
- Pro: elevations + BOM preview + more render/video
- Studio: approvals + commercial continuity + team workflows
- Brand: product/catalog/quotation workflows
- Real Estate: unit packages + presentation/sales workflows

---

## 5. Final Pricing Strategy Rule

Users should feel they are paying for:
- speed
- continuity
- accuracy
- production readiness
- fewer handoff errors

not only for “AI generations.”
