# 56 — Smart Implementation Plan from Coohom, BuildFlow, Naadi, and Service Studios

## Purpose

This document defines exactly how StudioOS should implement the strongest ideas from the newly scanned competitor set **smartly**, meaning:
- in the right order
- without bloating the product
- without breaking geometry-first architecture
- without copying irrelevant parts

---

## 1. The Four Capability Pillars to Merge

## Pillar A — Coohom-style design and render convenience
StudioOS should match:
- easy floor planning
- 2D → 3D progression
- model library usefulness
- fast first visual output
- AI-assisted design exploration

## Pillar B — BuildFlow-style workflow intelligence
StudioOS should match:
- high-level workflow orchestration
- AI-powered step routing
- export-focused pipeline thinking
- brief-driven automation

## Pillar C — Naadi-style design-to-manufacturing continuity
StudioOS should match:
- modular production awareness
- structured BOM logic
- future procurement and workflow mapping
- API-first integration design

## Pillar D — Service-studio storytelling quality
StudioOS should match:
- premium visuals
- client-facing presentations
- polished render boards
- premium framing of design proposals

---

## 2. Correct Build Order

## Phase A — sharpen the current core
Before adding more surface area, strengthen the current backbone:
- overlay editing maturity
- reviewed geometry confidence
- room/wall/opening accuracy
- stable scene patching and outputs
- drawing and BOM continuity

This is mandatory because everything else depends on trusted geometry.

---

## Phase B — implement the five workflow surfaces fully
Build and polish:
1. Smart Project
2. Quick Generate
3. Photo Edit
4. Quick Layout
5. Design Product

These should become the top-level workflow system.

---

## Phase C — add the tool hub / AI Studio
Once workflows are real, expose reusable specialist tools:
- material change
- mood shift
- camera angle change
- layout-to-3D
- exploded diagram
- product BOM
- quick elevation tools

The tool hub must route into real workflows and project states.

---

## Phase D — add commercial and library depth
Implement:
- catalogs
- vendor/brand entities
- material libraries
- finish packages
- presentation packages
- quote-building structures

---

## Phase E — add AURA-style agent orchestration
When the workflows are strong enough, layer in:
- routing agent
- plan review agent
- layout assist agent
- style/material agent
- render recommendation agent
- documentation agent
- budget optimization agent

---

## 4. Smart Exclusions

Do not waste effort trying to copy everything.

### Exclude or defer
- smart-home control stack as a core feature
- educational/news site structures as primary roadmap work
- generic marketplace behavior without scene linkage
- full IFC/BIM orchestration that over-expands beyond interior-first focus
- service-agency portfolio pages as if they were product capability

---

## 5. Product Modes to Introduce Carefully

## Mode 1 — Designer mode
This is the primary mode now.
Focus here first.

## Mode 2 — Brand / manufacturer mode
Introduce after stronger product/module and BOM systems exist.

## Mode 4 — Catalog mode
Introduce once product/material/furniture/tv uits data is reliable and useful.

---

## 6. Final Implementation Rule

The smart implementation path is:

1. strengthen canonical geometry core
2. complete the 5 workflow surfaces
3. expose specialist tools through a tool hub
4. deepen catalogs, presentations, and commercial layers
5. layer AURA-style intelligence on top
6. expand toward brand/developer/manufacturing integrations

That is how StudioOS can absorb the best of all these competitors without turning into a confused feature dump.
